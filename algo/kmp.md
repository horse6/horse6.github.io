## kmp算法
考虑模式字符串（pattern string），匹配字符串的时候如果前j个字符匹配了，到j+1个的时候，如果不匹配，如何把这个信息用起来？
暴力的解法里面是直接把pattern往后移一位，相当于把这个信息丢了，kmp算法就解决的是这个问题。

## next数组
kmp算法里面最关键的就是计算一个next数组，它是对pattern预处理得到的数组。
考虑pattern字符串`abcaba`,计算它的**每个前缀子串**的**相同前后缀子串**的最大长度：前缀子串是`a`,`ab`,`abc`,`abca`...，其中`abca`相同前后缀子串的最大长度是1（`a`）。  
这样可以从`abcaba`得到数组：`0 0 0 1 2 1`,在前面加上`-1`,去掉最后一个数`1`,就得到  
next = `-1 0 0 0 1 2`
我们定义 next[j] 为：当模式串第 j 位字符与文本串失配时，下一步应该用模式串的第 next[j] 位字符与当前文本串字符对齐继续比较。
next[0] = -1，表示第一个字符就失配时，模式串整体右移一位，文本串指针前进一位。
对于 j > 0，next[j] 的值等于 子串 P[0..j-1] 的最长相等真前后缀的长度。
例如，若 next[5] = 2，说明 P[0..4] 的前 2 个字符等于后 2 个字符，当第 5 位失配时，我们可以将模式串的前 2 个字符对齐到已匹配的文本串的后 2 个字符上。
实际计算中使用动态规划来进行计算，在匹配的时候使用的方式其实和计算next数组密切相关，搞清楚next的计算之后写匹配算法就很简单了。
计算 next 数组是一个动态规划的过程。我们用两个指针 i 和 k：  
- i 从 0 开始遍历模式串，表示当前正在计算 next[i+1]。
- k 表示当前已匹配的前缀长度，即 P[0..k-1] 已经与 P[i-k..i-1] 相等。
```python
def get_next(p):
    m = len(p)
    nxt = [-1] * m          # next[0] 固定为 -1
    i, k = 0, -1            # i 从 0 开始，k 初始为 -1
    while i < m - 1:        # 只需要计算到 m-2，因为 next[m-1] 会在循环内算出
        if k == -1 or p[i] == p[k]:
            # 如果 k 为 -1（开头）或当前字符匹配，则扩展匹配长度
            i += 1
            k += 1
            nxt[i] = k
        else:
            # 不匹配时，将 k 回退到之前的最长前后缀位置
            k = nxt[k]
    return nxt
```
上面代码需要注意的是while循环里面的两个分支，后一个分支只通过已有结果找[0...i-1]数组可能会匹配长度，然后到下一次进入while后在if分支判断是否匹配，直到匹配，
或者nxt[k]==-1（说明此时[0...i]的首位都是不一样，nxt[i+1]自然应该等于0）
